# S3-compatible API

Stohr exposes an S3-compatible API at `/s3`. Point any AWS client at it and it works.

## Mint an access key

In the web UI: **Settings → Developer → New access key**. The secret is shown **once** at creation — copy it immediately.

Or via the JSON API:

```sh
curl -X POST https://stohr.io/api/me/s3-keys \
  -H "Authorization: Bearer $STOHR_TOKEN" \
  -H "content-type: application/json" \
  -d '{"name":"laptop"}'
# → { "id": 1, "access_key": "AKIA…", "secret_key": "shhh…", "name": "laptop", "created_at": "…" }
```

## Use it

The bucket is your **username**. The key is a slash-separated path inside your stohr storage — Stohr auto-creates folders for path components.

```sh
export AWS_ACCESS_KEY_ID="AKIA…"
export AWS_SECRET_ACCESS_KEY="shhh…"
export AWS_DEFAULT_REGION="us-east-1"

# Upload
aws --endpoint-url https://stohr.io/s3 \
  s3 cp report.pdf s3://wess/reports/q4/report.pdf

# List
aws --endpoint-url https://stohr.io/s3 \
  s3 ls s3://wess/reports/q4/

# Sync a directory
aws --endpoint-url https://stohr.io/s3 \
  s3 sync ./build s3://wess/builds/v1.2.3/

# Download
aws --endpoint-url https://stohr.io/s3 \
  s3 cp s3://wess/reports/q4/report.pdf ./

# Delete
aws --endpoint-url https://stohr.io/s3 \
  s3 rm s3://wess/reports/q4/report.pdf
```

## Supported operations

| HTTP | meaning |
| --- | --- |
| `PUT /s3/<user>/<key>` | upload object (creates folder hierarchy from key path) |
| `GET /s3/<user>/<key>` | download object |
| `HEAD /s3/<user>/<key>` | metadata + existence |
| `DELETE /s3/<user>/<key>` | remove object |
| `GET /s3/<user>?prefix=<p>` | list (V2) — flat list of all matching keys |

## Quirks

- **Quota** — uploads count against your tier's `storage_quota_bytes`. Over-quota PUTs return `413 EntityTooLarge`.
- **Folders are auto-created** from key path components. `PUT s3://wess/a/b/c.txt` creates folders `a` and `a/b` if they don't exist.
- **Versioning** — re-uploading to the same key archives the previous live version (same as the regular API). The S3 `VersionId` parameter is currently ignored.
- **List** returns a flat list with no `CommonPrefixes` — folder simulation via `delimiter=/` is on the roadmap.
- **Multipart upload** isn't implemented yet. Files larger than your droplet's RAM may OOM the api container; for huge uploads, prefer the regular `/api/files` multipart endpoint or wait for streaming.
- **Sigv4** is required. `UNSIGNED-PAYLOAD` (the AWS CLI default for HTTPS) and pre-hashed bodies both work.

## Revoking a key

Settings → Developer → Revoke (or `DELETE /api/me/s3-keys/:id`). Anything using that key gets `403 InvalidAccessKeyId` immediately on the next request.
